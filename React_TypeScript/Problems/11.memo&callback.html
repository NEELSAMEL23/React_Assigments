🧩 React.memo & useCallback Optimization Challenge

📌 Problem Title:

Avoid Unnecessary Child Re-renders Using React.memo and useCallback

🧾 Context:

You're building a React app with a parent and child component. The parent maintains two state variables:

  • parentCounter — updated via a button in the parent
  • childClicks — updated via a button in the child component

Currently, when you update the parent state, the child component also re-renders — even though its props didn't change. This needs to be optimized.

🎯 Objective:

  • Use React.memo to memoize the child component
  • Use useCallback to memoize the function passed to the child
  • Log re-renders with console.log() in both components

📋 Requirements:

Parent Component (App.jsx):

  • State 1: parentCounter — incremented on button click
  • State 2: childClicks — updated by child button click
  • Pass a memoized handleChildClick to the child using useCallback
  • Log "🟦 Parent rendered" on each render

Child Component (Child.jsx):

  • Props: onClick (function), clicks (number)
  • Log "🟨 Child rendered" when the component renders
  • Wrap the component with React.memo

✅ Expected Behavior:

Action Should Parent Re-render? Should Child Re-render?
Click "Increment Parent" ✅ Yes ❌ No
Click "Click Me" (Child) ✅ Yes ✅ Yes

🧪 Console Logs:

  • Clicking parent button → logs 🟦 Parent rendered
  • Clicking child button → logs 🟨 Child rendered and updates count

💡 Hint:

Use useCallback(() => {...}, []) to prevent function recreation and prop change in the child.

Tip: Use React DevTools Profiler to confirm render behavior visually.
