Use of Chakra UI is mandatory

Overview

  • Create a Redux store to manage the state of a list of users fetched from an API.
  • Use redux-thunk middleware to handle asynchronous actions.

Fetch Data:

  • Fetch user data from the following API:
  • https://jsonplaceholder.typicode.com/users
  • Dispatch an action to save the fetched user data in the Redux store. Routing:
  • Use react-router-dom to set up two routes:
  • /: Displays a list of user names.
  • /user/:id: Displays detailed information about a selected user (e.g., name, email, phone, and address). Components:

HomePage:

  • Show a list of user names. Each name should be a link to the /user/:id route.

UserDetailsPage:

  • Fetch the details of the selected user from the Redux store using the id from the route parameter. Bonus:
  • Show a loading spinner while the API data is being fetched.
  • Display a "Back to Home" button on the user details page to navigate back to the home page.

Expected Features

  • Use React-Redux for state management.
  • Use redux-thunk for asynchronous API calls.
  • Use react-router-dom for routing between pages.
  • The UI should update dynamically based on the data from the Redux store.
